Demo App

Diese Demo-App dient zum Testen des Rest API Authentifizierungsmoduls für sysHUB. Entweder muss das Modul aus npm installiert werden oder aber aus einem Entwicklungs-Ordner verlinkt werden. Siehe mehr Infos in der Readme in diesem Repository.

Diese Demo-Seite stellt Testfälle bereit welche die das Auth-Modul implementiert. Für eine Dokumentation der Endpoints im sysHUB bitte das Swagger UI nutzen.

Das Auth-Modul enthält auch die Methoden delete(), patch() und put(), um alle Arten von HTTP-Methoden zu unterstützen, die die sysHUB Rest API verwenden kann. Sie funktionieren genau wie get() und post(), daher werden diese hier nicht mit extra Beispielen gezeigt.

Voraussetzungen

Damit die Tests dieser Seite funktionieren können, müssen folgende Voraussetzungen erfüllt sein:

  1. sysHUB Expertenkonfig
    1. Authorisierungs-Server (OAUTH)
      1. Erlaubte Origins: Entweder * oder die Hostname dieser Seite muss enthalten sein, ansonsten kommt es zu CORS-Fehlern.
      2. Clients: Ein eigener Client ist angelegt. Client Id und Client Secret müssen in diesem Projekt in die Dateien src/environments/environment.ts und src/environments/environment.development.ts eingetragen werden.
    2. API-Server (Basic Auth)
      1. Erlaubte Origins: Entweder * oder die Hostname dieser Seite muss enthalten sein, ansonsten kommt es zu CORS-Fehlern.
      2. Authentication Provider: Ein eigener Provider ist angelegt. Provider, Benutzername und Passwort müssen in diesem Projekt in die Dateien src/environments/environment.ts und src/environments/environment.development.ts eingetragen werden. Bei diesem Provider muss die Checkbox Basic Auth aktiviert sein.
    3. Scope: Sowohl bei OAUTH als auch Basic Auth kann ein Scope angegeben werden, das standardmäßig mit public befüllt ist. Alternativ kann private oder private;public angegeben werden. Im Swagger UI ist rechts oben ein Dropdown für REST-API und REST-API internal. Letzteres enthält viele Endpoints die nur mit dem Scope private zur Verfügung stehen. Ungültige Werte führen zu fehlendem Zugriff auf die REST API.
  2. sysHUB Benutzerverwaltung
    1. OAuth: Bei OAuth gibt der Benutzer seinen Benutzernamen und sein Passwort ein. Der Browser sendet diesen zusammen mit Client Id und Passwort an den Server und erhält einen temporär gültigen Token. Dieser Token wird mit jeder weiteren Anfrage gegen die Rest API mitgesendet. Der Benutzer muss also angelegt und aktiv sein und muss alle Berechtigungen erhalten, welche gem. Swagger für die aufzurufenden Endpoints erforderlich sind.
    2. Basic: Bei der Basic Authentifizierung wird ein Benutername und ein Passwort mit jeder Rest API Anfrage an den Server mitgesendet. Es ist kein dedizierter Login erforderlich. Dieser Benutzer muss also angelegt und aktiv sein und muss alle Berechtigungen erhalten, welche gem. Swagger für die aufzurufenden Endpoints erforderlich sind.
